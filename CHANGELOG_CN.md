# 更新日志

## [2.1.1] - 2026-07-22

> ℹ️ **此版本修复了一个长期的扣费问题。** 自上游 Kelivo v1.1.6 起，点击"停止"从未真正关闭底层 TCP 连接。供应商无法收到取消通知，即使支持流式取消的模型也可能在后台静默继续生成，导致意外的 Token 消耗和多扣费。**建议所有用户升级。**

### 修复

- 停止按钮现在会强制关闭底层 TCP 连接，确保供应商立即收到取消通知，杜绝导致多扣费的后台静默生成（此问题自上游 Kelivo v1.1.6 起存在）(#79)
- 更新关于页面的 GitHub 链接和更新检查地址到 cuplivo/cuplivo (#80)

## [2.1.0] - 2026-07-21

> ⚠ **本版本包含重要的备份恢复修复。** 在 1.5.0–2.0.2 版本中，"智能合并"并不会将备份文件中的助手合并到本地数据库，现已修复为正确的合并逻辑。
>
> v2.0 引入了**安卓平台特有功能**"Ta 的来信"（Proactive Care）。可能需要额外权限，但不授予这些权限不影响其他功能的使用。
>
> v2.1 开始引入**技能**（**Skill**）机制。首版功能暂不完全，将在后续版本逐步补全。

### 新增

- SKILLS 机制 — 基于文件系统的技能存储、从文件/手动导入、助手绑定、`load_skill` 工具、备份支持

### 修复

- 智能合并现在能正确从备份文件恢复助手（影响 1.5.0–2.0.2 版本）
- 将助手数据从 SharedPreferences 恢复到 SQLite（防止数据丢失）(#74)
- Multi-AI anchor 系统改用 `groupId` 而非 `messageId`，修正线程解析 (#72)
- 重命名 `userImagePaths` 为 `userMediaPaths`；修复 Responses API 中的 Office 文档处理 (#66)

## [1.9.0] - 2026-07-19

> ⚠️ **图片生成用户必须更新**
> 
> 自 1.4.1 版本切换图片响应为 base64（`prefer b64_json`）以来，
> 基于正则表达式的图片引用扫描器在大尺寸 base64 负载上会出现灾难性回溯，
> 导致任何图片生成对话的第二轮出现栈溢出。1.9.0 版本将正则表达式替换为
> 线性 indexOf 扫描。

### 新增

- PDF/Office 文件附件：直接上传 PDF、Word、Excel 和 PowerPoint 文档，附带文档处理配置(#26)

### 修复

- Multi-AI 重试矩阵：上下文截断、版本选择时机和防护提示(#54)
- MCP：在速率限制错误(429 / -32106)上跳过心跳重连(#53)
- 大尺寸 base64 图片不再导致正则表达式栈溢出 — 替换为 indexOf 扫描(#55)

## [1.7.2] - 2026-07-19

> ⚠️ **使用 SVG 预览的 1.7.0–1.7.1 用户必须更新**
> 
> 之前的版本在 LLM 流式输出 SVG 代码块时崩溃：flutter_svg 的
> 隔离解析器在流式输出期间对部分/无效的 XML 块重复触发 `compute(encodeSvg)`，
> 导致隔离区风暴。1.7.2 增加了流式防抖（360 ms 活跃 / 220 ms 稳定）、
> 流式传输时自动切换到代码标签页、实施 1 MB 大小限制，并增加了
> 错误回退。SVG 预览用户应立即升级以避免随机应用崩溃。

### 新增

- 桌面端比对视图每页显示 2 个模型列，取代单卡片滑动(#50)

### 修复

- SVG 预览在 LLM 流式输出时的隔离区风暴崩溃(#46)
- 线程锚点在 `dropThread` 后丢失导致自动采纳失败(#51)

## [1.6.0] - 2026-07-13

> 💡 **更新说明**
> 
> 此版本引入了**记忆模式切换器** — 每个助手可独立选择
> **自动注入**（记忆始终注入系统提示词）或**按需工具**（仅在需要时通过 `read_memory` 工具读取记忆）。
> 工具模式保持系统提示词稳定，显著提高 API 缓存命中率并降低延迟。

### 新增

- 记忆模式切换器 — 每个助手可在自动注入和按需工具模式之间切换
- `read_memory` 工具用于按需记忆检索（工具模式下）

### 变更

- 记忆系统现支持按需（工具）模式：不再总是将所有记忆注入系统提示词，助手可在需要时通过工具读取记忆。
  这保持了系统提示词的稳定，极大提高 API 缓存命中率
- 提取了 `_cleanupStreamingError` 工具函数；修复了复制粘贴的日志标签错误
